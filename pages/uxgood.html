There are many ways to fix the pre-filled text issue. Below, I have proposed one method of fixing the issues. This style can be modified to clarify many additional questions throughout the form. As a result, the textboxes located on the form will offer a much better user experience.

Behind the scenes, there are limits and constraints for the users. However, typing out the number into an already cleared field is easier than the previous problem presented. The "Confirm" and "Reset Form" buttons would be located at the bottom of the page for the user to save or reset the information that has been entered. The previous buttons on the intial interface implied that the data on the page was not cleared or reset if the user left the page. Implementing a "Reset Form" button can be very helpful for the user versus the previous "Close" button.




This design improves the user experience because the user does not have to delete any text. The user can automatically add the numbers into the textbox without deleting the pre-existing numbers. If there is an error, the user can easily reset the fields located on the form. If the user wants to confirm the number the information they have entered, the Confirm button allows an easy route to confirm the selection.
